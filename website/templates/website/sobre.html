Sobre

Tecnicas usadas:

Parte 1: Na criação deste website, sobre futebol, utilizei CSS para estilização de todas as páginas, header, nav, main, aside, footer, imagens, texto, títulos, animações e responsividade do site e dos mesmo. Utilizei CSS Grid para estilizar/formatar a estrutura dos 2 layouts que criei/utilizei, e Flex no display de alguns conteúdos. Foram também utilizados diversos ícons, animações, fonts de letras, uma imagem Wordlcloud, algumas imagens e vídeos para tornar o site mais dinâmico. O formulário criado, na forma de um quizz, sobre o mais diverso conteúdo do site envia as respostas dos utilizadores para um e-mail. Além, das técinas e propriedas já enunciadas, utilizei várias técnincas do HTML e CSS asseguir enunciadas/tabeladas.

Parte 2: Utilizando a base feita na parte 1 do projecto, re-criei o mesmo site, mas desta feita, em Django. Para isso utilizei as várias compenentes deste mesmo sistem. URLS.py para gravar os caminhos(PATH) a serem prosseguidos pelo website. URLS na pasta config. contendo o caminho a ser feito pelo browser para ir ao encontro de outro URLS.py, mas desta feita, com o caminho específico de cada app contida no projecto. No URLS.py(na pasta da app correspondente), contem agora o caminho específico para um hmtl ou outro da app pedida, identificando a "função" dentro da página VIEWS.py pela qual este recurso pedido deverá ser renderizado antes de ser enviado como resposta. No VIEWS.py(da app corresponde), recebe um request do utilizador, ou browser, e através da "função", previamente identificada pelo URLS.py, que tem contida faz algumas operações necessários como editar a base de dados e retorna a renderização dessa página de acordo com o template indicado. Na pasta template(de cada app), tenho inserido uma base.html que serve como "molde" para todos os outros ficheiros template.html de forma a apresentarem a mesma "raiz". Estes(templas.html) renderização a página. Em MODELS.py defeni os modelos(classes) para as base de dados que seram necessárias para o website, bem como e tipo e variávis. Aqui utlizei Primary Key, em Contacto, Foreign Key, em Jogadores e Comentários, de forma a ter 3 classes interligadas dependes da class Contacto, ou seja, apagando um objecto da class contacto seram apagados todos os objectos a si associados(on_delete=models.CASCADE). Em ADMIN.py estão registrados estes models. Em FORMS.py programei vários formulários para inserir dados em base de dados do website, devidamente acedidos por um admin. Foi feita também uma página contactos, apenas acessível a um utilizador autenticado, que permite fazer operações CRUD sobre estes dados (listar, editar e apagar contactos).

Parte 3: Fiz uma SinglePage, com utlização de JavaScript(ficheiro js), que funciona como uma base de dados dos jogadores. Para isso implementei os seguintes butões: *Butão 1-lista os jogadores registrados e obtidos pela API inserida em VIEWS.py. -Este butão retornm "páginas" formadas pelo js. Ou seja ao carregar nestes butões(onclick()) os mesmos através de Handlers fazem operações DOM ao registo da API previamente gravada de forma a retornar os dados necessários ao pedido desse butão. Após receber estes dados uma funçao do js cria um array de HTML em que cada posição irá registrar uma página(html) com a informção contida na API. Esta operação e recursiva até que todas as posições na API terem uma página(html) criada e introduzida no array de HTML. Renderizadas por uma funçãoTemplate() gravada no js. Este array HTML é retornado depois como uma página com todos os jogadores listados na API para um div(com context id == à seção section pedida) que esta previamente escondido(hidden) como todos os outros, até ser necessário. Por fim cada jogador listado, nesta página, tem um butão que faz basicamente o mesmo anteriormente descrito através de outra funçãoTemplate() com toda a info da posição do API específico que pretendemos. *Butão 3-lista os jogadores a espera de serem inseridos pela API no registo, que foram inseridos pelos utilizadores do site. Correndo todos os Objectos do tipo Jogador grvados na base de dados. *Butão 4-retorna a página para a Homepage do website. *Butão 5, 6, 7-faz manipulações do DOM com JavaScript de forma a mudar a cor de fundo do site(5), cor das letras(6) e style das letras(7).
